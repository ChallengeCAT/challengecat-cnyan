# Contributing

Thank you for improving CNYAN / ChallengeCAT.

## Principles

- Keep the repository project-first.
- Do not add personal identity information.
- Keep copy clear, natural, and multilingual where useful.
- Avoid financial promises or guaranteed-return language.
- Prefer small, reviewable changes.

## Suggested Areas

- UI polish
- Solana token documentation
- multilingual copy
- accessibility improvements
- animation performance
- metadata hygiene

## Local Workflow

```bash
npm install
npm run dev
npm run build
```

## Copy Rules

Use:

- CNYAN
- ChallengeCAT
- the project
- the community

Avoid:

- personal names
- founder claims
- private contact details
- private wallet ownership claims
- location-identifying details
