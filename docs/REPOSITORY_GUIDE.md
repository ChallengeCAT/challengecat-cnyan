# Repository Guide

This repository is designed to look credible, organized, and useful for a Solana token/community demo without exposing personal identity.

## Recommended Public Description

```text
CNYAN by ChallengeCAT — Solana community challenge demo with bounty maps, proof submission, route intelligence, and CNYAN/SOL reward flows.
```

## Recommended Topics

```text
solana
spl-token
metaplex

multilingual
```

## What Not To Add

- Personal names
- Personal emails
- Private wallet ownership claims
- Location details
- Founder doxxing language
- Promises of profit or guaranteed liquidity

## What To Add When Publishing

- Pin this repository on the ChallengeCAT GitHub profile.
- Add the CA to the repository description only if it fits.
- Use the `.github/profile/README.md` file if creating an organization profile repository.
- Keep screenshots and demo links project-focused.
